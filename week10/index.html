Client-side Form Validation

  • Ensures that all required form controls are filled out before the form is submitted
  • Allows the user to fix the data right away.. Doesn't feed in bad information to the backend
  • Server-side validation is once the data is passed through. Clien-side is before it is submitted
  • Goal: Get the data in the right format, protect our users' data, protect our program
  • Types of form validation:
    • Built-in form validation: uses html 5. Not as customizable as JS
    • JavaScript: coded in JS, completely customizable but needs to be created by scratch or pulled in from a library

    • Built in Form Validation:

  • Use validation attributes on form elements
  • attributes: required, minlength, maxlength, min, max, type, pattern
  • If elemnt is valid it will match a :valid CSS class which allows you to apply specific style to valid elements
  • When element is invlaid it will match :invalid CSS class. Browser will block the code from being sent.
  • Link to a built in form validation example
  • After trying both of those out, I feel like built in form validation is much easier. But I understand that there is a lot more customizable options with JS.
